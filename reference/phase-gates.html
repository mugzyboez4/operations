Framework
RCA → RUNS · CONSULTED · ACCOUNTABLE
How roles work across departments. The 3-letter version of RACI — minus the I, because the system handles informing. Each row in every phase gate has exactly one A. If you can't pick one, that's the conversation that needs to happen.
R · Runs It
  • Does the work. Hands on keyboard. In the room. Producing the thing.
  • Multiple R's per row are fine — that's shared labor, not ambiguity.
  • In these checklists, R is implicit: the dept the row sits under is the runner.
C · Consulted
  • Has input before the work ships. Two-way.
  • Their perspective shapes the output — they're not just told after.
  • Use sparingly. Over-consult and you never ship.
A · Accountable + QC
  • Owns the outcome. If it falls through, this is whose problem it is.
  • Owns quality. The A is the last set of eyes before it goes out the door.
  • Exactly one per row. No exceptions. Two A's means nobody is A.
  • When something ships with a quality issue, you look at the grid, find the A, conversation is framed.
No I · The System Handles Informing
  • Airtable views, EOD drops, recaps, TEMPO — that's the notification layer.
  • No human role for "told after." The infrastructure does that automatically.
  • This is the Campaign Ops thesis: the system informs; people run, consult, own.
How this maps to the gates: In each phase gate, every row sits under a department section. That section is the default A. Where the chat thread or operational reality says otherwise, the override is tagged on the row (Workability → Campaign Ops; Clearances Confirmed → Business Affairs; etc.). Items under Campaign Ops — Cross-Dept are gate connective tissue — Campaign Ops is A there because no other single department owns them.
