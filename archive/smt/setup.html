SMT Dashboard
Setup Guide

How to wire the live Airtable data behind /smt/live.html. ~10 minute setup.

1. Add 2 fields to the Artists table

In Airtable (base appLwUF3H4KjtiRdW, Artists table), add two new fields:

Field nameTypePurpose
SMT BriefingLong textJSON blob with phase / project / big stat / results / issues for this week's card.
SMT OrderNumberDisplay order on the dashboard (1–6 typically). Empty = excluded.

Field names must match exactly — the function references them by name.

2. Populate the 6 SMT artists

For each artist on the dashboard, set SMT Order (1–6) and paste a JSON blob into SMT Briefing:

{
  "project": "BROWN",
  "project_meta": "Album · Released 5/7",
  "big_stat": {
    "value": "4M+",
    "label": "Projected streams · Week 1"
  },
  "results": [
    ["60–70K", "Album-equivalent range tracking · Week 1"],
    ["45K", "Beat the 11/11 benchmark"]
  ],
  "issues": [
    "Mixed reviews on the back half of the album",
    "Physical media plan for tour sales — still to lock"
  ]
}

Field-by-field

  • project (string) — main title shown under artist name. Required.
  • project_meta (string) — appended after ·. Optional ("Album · 6/26").
  • big_stat (object or null) — the chartreuse number card. Set to null to skip.
  • results (array of [value, label] pairs) — left number, right description. Empty array hides the section.
  • issues (array of strings) — bullet points with red dots. Empty array hides the section.

Phase comes from the existing Phase field on Artists (Prep/Plan/Announce/Release/Review). The function remaps Announce→LAUNCH and Review→ACTIVE per Jordan's 5/12 directive. No need to duplicate phase in the JSON.

3. Set env vars in Netlify

The function reuses the existing roster credentials. If /api/roster works, this works too. The env vars used:

  • AIRTABLE_TOKEN — Airtable PAT with read scope on the base
  • ROSTER_PASSWORD — same password gate as the artists page

4. Embed in Airtable Interface

In Airtable Interface Designer:

  • Open the Executive Interface (or create a new one)
  • Add a Webpage element to the front page
  • Paste https://operations.mugzyboez.co/smt/live.html
  • Save. Users will see the password prompt, enter the roster password, and the dashboard renders inside the Airtable Interface.

If Airtable rejects the embed with a "this site can't be embedded" error: the _headers file in this repo allows iframing from Airtable origins. If it still fails, fallback to a Button element pointing to the URL (opens in a new tab).

5. Troubleshooting

SymptomCauseFix
